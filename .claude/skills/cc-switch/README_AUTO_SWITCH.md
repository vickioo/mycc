# cc-switch 智能自动轮替框架使用指南

## 🎯 功能概述

cc-switch 智能自动轮替框架是一个高级的模型管理工具，具备以下核心功能：

- **实时健康监控**：持续检测各模型的可用性和响应性能
- **智能故障切换**：当当前模型出现问题时自动切换到最佳替代模型
- **性能优化**：基于响应时间和成功率选择最优模型
- **完整日志记录**：详细记录所有切换事件和系统状态

## 🚀 快速开始

### 1. 分析当前模型配置
```bash
cd /data/mycc
python3 ../mycc/.claude/skills/cc-switch/scripts/auto_switcher.py --analyze
```

### 2. 执行一次性健康检查
```bash
python3 ../mycc/.claude/skills/cc-switch/scripts/auto_switcher.py --once
```

### 3. 启动持续监控
```bash
python3 ../mycc/.claude/skills/cc-switch/scripts/auto_switcher.py --monitor
```

### 4. 使用守护进程管理
```bash
# 启动守护进程
../mycc/.claude/skills/cc-switch/scripts/cc-switch-daemon.sh start

# 查看状态
../mycc/.claude/skills/cc-switch/scripts/cc-switch-daemon.sh status

# 停止守护进程
../mycc/.claude/skills/cc-switch/scripts/cc-switch-daemon.sh stop
```

## ⚙️ 配置说明

### 监控参数
- **检查间隔**：默认300秒（5分钟）
- **健康阈值**：成功率≥80%，连续失败次数<3次
- **响应时间窗口**：最近10次请求的平均值

### 优先级策略
系统按照以下优先级自动选择模型：
1. GLM4.7（NVIDIA API）- 当前最优选择
2. Antigravity Manager - 本地代理服务
3. Minimax M2.1 - 中文优化模型
4. KIMI - 知识密集型任务
5. Google AI Studio系列 - 备用选项

## 📈 监控指标

### 健康状态分类
- `healthy`：模型正常响应
- `unhealthy`：模型出现故障
- `unknown`：尚未进行健康检查

### 关键指标
- **成功率**：近期请求的成功比例
- **平均响应时间**：请求处理的平均耗时
- **连续失败次数**：持续失败的请求数量
- **最后错误信息**：最近一次失败的具体原因

## 🔧 故障处理机制

### 自动切换条件
当满足以下任一条件时触发自动切换：
- 当前模型状态变为`unhealthy`
- 成功率低于80%阈值
- 连续失败次数达到3次

### 切换决策逻辑
1. 优先选择优先级列表中的健康模型
2. 在健康模型中选择响应时间最短的
3. 如果优先模型都不可用，则激活备用模型
4. 记录切换事件用于后续分析

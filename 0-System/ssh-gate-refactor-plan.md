## 当前问题

1. **databasei 配置臃肿** - 7 个 LocalForward 绑定在跳板机别名上

## 后台服务管理方案

### 仿 Termux sv-style 设计

```
/root/.local/sv/
├── ssh-gate/          # SSH 端口转发服务
│   ├── run            # 执行脚本
│   ├── conf           # 配置文件
│   └── log/           # 日志目录
├── cc-clash/          # CC Clash 隧道 (可选)
│   ├── run
│   └── conf
└── watchdog/          # 健康检查守护进程
    ├── run
    └── conf

/root/bin/sv/
├── sv-start           # 启动服务
├── sv-stop            # 停止服务
├── sv-restart         # 重启服务
├── sv-status          # 查看状态
└── sv-log             # 查看日志
```

### 服务脚本特点

1. **轻量无依赖** - 纯 Bash，不需要 Node/Python
2. **PID 文件管理** - 精确控制进程
3. **自动日志轮转** - 避免日志暴增
4. **健康检查** - 内置存活检测
5. **开机自启** - 可集成到系统启动

## 🔧 技术实现

### 前端
- **重试逻辑**: 指数退避算法
- **状态检测**: Fetch API + AbortSignal
- **Toast 通知**: 动态创建 + 自动移除
- **心跳检测**: setInterval 10 秒

### 后端
- **系统状态 API**: `/api/system-status`
- **健康检查**: `/api/health`
- **监控页面**: `/monitor`

### 数据流
```
用户请求
  ↓
检查连接状态
  ↓
发送请求 → 失败 → 重试 (最多 3 次)
  ↓           ↓
成功       达到最大重试
  ↓           ↓
显示回复   显示错误 + 重试按钮
```

## 🎯 使用指南

### AI 对话稳定性

1. **自动重试**: 请求失败自动重试，无需手动操作
2. **手动重试**: 点击错误消息的"🔄 重试"按钮
3. **切换 Agent**: 如当前 Agent 离线，切换到其他可用 Agent

### 系统监控

1. 访问 http://localhost:8772/monitor
2. 查看健康度评分
3. 检查各服务状态
4. 查看实时消息
5. 点击"刷新状态"立即检查

### 通知系统

- **成功** (绿色): 回复已生成、已复制等
- **警告** (黄色): Agent 离线、重试中等
- **错误** (红色): 请求失败、服务异常
- **信息** (蓝色): 系统消息
